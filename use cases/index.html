# Сценарії використання

ID SignUp
Назва Зареєструвати користувача (Sign In)
Учасники Користувач, Система
Передумови Користувач не має облікового запису у системі
Результат Створений обліковий запис користувача
Виключні ситуації NullReferenceException - Користувач не заповнив усі обов'язкові поля
AccountAlreadyExistException - Обліковий запис с такими данними вже існує
NotStrongPasswordException - Користувач увів пароль, який не відповідає всім вимогам "надійного паролю" (A-Z, a-z, 0-9, !-*, мінімум 8 символів)
Основні сценарії 1. Користувач заходить на сторінку створення облікового запису
Користувач ввів данні у всі поля
2. Користувач натискає кнопку "Реєстрація" (NullReferenceException, NotStrongPasswordException)
3. Перевірка наявності облікового записи з ідентичними данними (AccountAlreadyExistException)
4. Створення облікового запису системою
5. Користувач переходить на головну сторінку системи під своїм обліковим записом
ID GoogleSignUp
Назва Зареєструвати користувача (Sign In) за допомогою GoogleApi
Учасники Користувач, Система
Передумови Користувач не має облікового запису у системі
Результат Створений обліковий запис користувача через інформацію, отриману з профілю користувача
Виключні ситуації InvalidCredentialsException - Помилкові облікові дані Google або доступ до облікового запису відмовлено
UserAlreadyExistException - Обліковий запис, створений за допомогою GoogleApi, вже існує в системі
Основні сценарії 1. Користувач заходить на сторінку створення облікового запису
2. Користувач натискає "Зареєструватися за допомогою облікового запису Google"
3. Користувач вводить дані облікового запису в GoogleApi (InvalidCredentialsException)
4. Система перевіряє наявність облікового запису користувача (AccountAlreadyExistException)
5. Система створює новий обліковий запис
6. Користувач переходить на головну сторінку системи під своїм обліковим записом

Результат Проблема користувача вирішена
Виключні ситуації NullReferenceException - не заповнені обов'язкові поля.
CancelTokenException - операцію скасовано користувачем

5. Адміністратор вирішує проблему та повідомляє користувачеві рішення
6. Користувач отримує рішення проблеми або сповіщення про відсутність відповіді від адміністратора (NoResponseException)
